# Parsing Hints by Language

Reference for identifying instance fields, statics, and getter/setter heuristics per language.

---

## Python

**Instance fields**: Any `self.x` — whether assigned in `__init__` or any other method.
Don't confuse with local variables (no `self.` prefix).

**Static/class fields**: Defined at class level without `self`, or decorated with `@classmethod` /
`@staticmethod`. Exclude from the field set.

**Getter/setter heuristic**: Methods decorated with `@property`, `@x.setter`, `@x.deleter`.
Also flag single-line methods like `return self.x` or `self.x = value` with no other logic.

**Constructors**: `__init__` only. `__new__` is a special case — exclude it too.

---

## Java

**Instance fields**: Declared at class level, no `static` keyword.

**Static fields**: Declared with `static`. Exclude.

**Getter/setter heuristic**: Methods named `getX()` / `setX(value)` / `isX()` with a body
that only reads or writes a single field (1–3 lines). Use name pattern + body simplicity together.

**Constructors**: Method with same name as the class, no return type.

**Note on inheritance**: `super.fieldName` — exclude. Direct `this.fieldName` or bare `fieldName`
(resolved to instance field by scope) — include.

---

## C#

**Instance fields**: Declared at class level, no `static`. Includes auto-property backing fields.

**Properties** (`get`/`set` blocks): Exclude entirely — these are the getter/setter equivalent.

**Static fields/methods**: Exclude.

**Constructors**: Method with same name as the class, no return type.

**Expression-bodied members** (`=> this.x`): Treat as getters — exclude.

---

## JavaScript / TypeScript

**Instance fields**: `this.x` assignments anywhere. In TypeScript, also class-level field
declarations without `static`.

**Static fields/methods**: `static` keyword. Exclude.

**Getter/setter heuristic**: `get x() { return this._x; }` / `set x(v) { this._x = v; }` syntax
— exclude. Also flag arrow-function methods that are one-liners accessing a single field.

**Constructors**: `constructor()` method.

**TypeScript-specific**: Private fields (`#field` or `private field`) are still instance fields —
include them.

---

## Ruby

**Instance fields**: `@variable` (instance variables). Not `@@variable` (class variables) and
not `$variable` (globals).

**Class variables** (`@@x`): Exclude.

**Getter/setter heuristic**: Methods generated by `attr_reader`, `attr_writer`, `attr_accessor`
— exclude all of them. Also exclude any method that is literally just `@x` or `@x = value`.

**Constructors**: `initialize` method.

---

## Go (Structs)

**Instance fields**: Fields on the receiver struct. If the method receiver is `func (r *Receiver)`,
then any `r.fieldName` is an instance field access.

**Static equivalent**: Package-level functions with no receiver — already excluded (not methods).

**Getter/setter heuristic**: Single-line methods returning or setting one field. Go convention
uses `Name()` / `SetName()` patterns — apply the same 1–3 line + single-field heuristic.

**Constructors**: `NewX()` constructor functions have no receiver — exclude (not instance methods).

---

## General Heuristic for Unknown Languages

When the language isn't listed above:

1. **Instance fields** = member variables on the object/struct that are not `static`, `class-level`,
   or otherwise shared across instances.
2. **Getter/setter**: Apply the heuristic — body ≤ 3 lines, accesses exactly 1 field, no branching.
3. **Constructor**: Method named after the class, or a well-known initializer convention for
   that language.
4. When uncertain, **include rather than exclude** — it's safer to over-count connections than
   to silently drop methods and undercount LCOM4.
